Parser System

Information:

The parser system is responsible for dynamically generating content to fill writing with words or even entire sentences that match the subject in question. This could be the size of a physical attribute, or words that a certain character type (or PC persona) would be likely to use. Arguments used in parser (and other) functions must be entered in the correct order as listed, or will not work properly.

[ ] : Indicates that the argument is optional two arguments included in a single bracket indicate that if one is used, the other must be used. This is standard notation, you can read more elsewhere.
T : This argument letter indicates target character. Typically this indicates which character the value should be based on. The argument values allowed vary, but are typically null for the PC, npcid for NPC, or the $activeNPC index.
